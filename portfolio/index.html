PROJECT TWO/

Media Campaign

CLIENT/

Nasmak PM

PROJECT OVERVIEW

I built a Media Campaign for Nasmak PM, an experimental electronic band, to help them attract younger audiences, grow their fanbase and have a stronger social media presence.

SKILLS USED

  • Photoshop
  • Trello
  • Research
  • Chat GPT
  • User Testing

DATE

9 September -

Research Process

"I initiated the research phase by developing research questions focused on understanding the band's audience and their preferences. From there, I conducted library research, analyzing articles and relevant sources to gather insights into audience demographics and interests. These findings shaped our media campaign strategy and guided creative decisions around content ideas”

Design Process

"To kick off the design process, I suggested that my teammates and I listen to Nasmak PM's music—a unique, improvised experimental electronic band—while creating mood boards. This approach allowed us to visually interpret how each of us connected with their distinctive sound, capturing the diverse emotional responses and energy we felt. These mood boards not only provided valuable insights into the band's avant-garde identity but also served as a key source of inspiration for our design work moving forward. Building on this, I used some of the images from the mood boards to create posters in Photoshop, experimenting with different design elements to capture the experimental essence of Nasmak PM's music and visually represent their unique style."

PROJECT THREE/

Development

CLIENT/

Nasmak PM

PROJECT OVERVIEW

I developed a web app which allowed Nasmak PM to create custom, interactive videos to be used on their social media platforms.

SKILLS USED

  • JavaScript
  • HTML
  • CSS

DATE

13 November 2024 -
7 December 2024

How I started

My teammate and I worked together on the Media Campaign and so we had a clear idea as to what we wanted to do for the development project. We both wanted to create something that would be useful to the client and help them combat their challenge which was reaching more audiences. So, we decided to create an app for them to create videos with the mascots which we thought of in the previous project for their social media platforms. With both of us having limited skills in JavaScript, we knew that creating this interactive app would be a challenge but we were determined to take on the challenge

Prototyping

I started by creating a basic prototype to serve as a proof of concept for our idea. I did this by creating a canvas with two shapes that could move around with the arrow keys. I created two separate elements, a blue square to represent the sprite with an closed mouth and a red circle to represent the mouth open. Then Yurick added the recording function and we went to Stan and Maikel for feedback.

Version Control & GIT

My teammate and I utilized Git throughout our project to ensure seamless collaboration and version control. We began by creating a shared repository on GitLab. This allowed both of us to have access to the project's codebase. After setting up the repository, we cloned it locally using Git, enabling us to work on the code independently while keeping everything synchronized.

We made sure that each commit message was descriptive and followed a clear format, like add sprite toggling functionality or resolve merge issue. This made it easy to review the history of changes and understand what each update was about. Occasionally, we encountered merge conflicts when both of us edited the same part of the code but with the help of research and ChatGPT, we were always able to solve the issues. Using Git's version history, we could always revert to an earlier state of the project if something broke. This gave us confidence to experiment without the fear of losing stable code.

This was one of my first time collaborating with someone using git and so there were places where I want to improve on. For example, we worked on the same branch for everything but after research I learned that this is not good practice and that in the future, we should have separate branches for specific features or tasks.
