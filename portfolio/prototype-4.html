Prototype One

We started off by splitting tasks in order to speed up the process. Yurick focused on getting the recording function to work while I created the canvas and focused on getting the charcters to move around with keys on the keyboard.

I started by creating a basic prototype to serve as a proof of concept for our idea. I did this by creating a canvas with two shapes that could move around with the arrow keys. I created two separate elements, a blue square to represent the sprite with an closed mouth and a red circle to represent the mouth open. Then Yurick added the recording function and we went to Stan and Maikel for feedback.

Explaining my code

Canvas API for Animation

CANVAS SETUP: The setupCanvas method uses the Canvas API (getContext('2d')) to enable 2D drawing and animations.

DRAWING AND ANIMATION: Shapes (square and circle) are dynamically rendered based on the state (showSquare). Smooth animation is achieved using the requestAnimationFrame in gameLoop.

DYNAMIC BEHAVIOR: Shapes change dynamically between square and circle depending on the space bar press/release (keydown and keyup). Arrow keys control the movement of the shape across the canvas.

Shape Rendering

SHAPE RENDERING: The square and circle share properties like position and speed, emphasizing reusability.

SWITCHING SHAPES: The showSquare variable determines whether to draw a square (fillRect) or a circle (arc).

BOUNDRY CONTROL: The update function prevents shapes from moving outside the canvas, using Math.max and Math.min

Key press handling

STAGE MANAGMENT WITH KEYS: The keys object tracks which keys are currently pressed.

RESPONSIVE MOVEMENT: Arrow keys adjust the position of the shapes in real-time, providing smooth control.

Prototype Two

In the next prototype, we took Stans advice and replaced our shapes with the sprite images. This gave us a better idea as to what the final app would look like as this allowed us to interact with the actual sprite character. Much of the code was similar to prototype one, only changed to work for the images and not the shape elements.

Explaining My Code

Replacing Canvas-Drawn Shapes with Images

The previous implementation dynamically drew shapes (a square and a circle) on the canvas using ctx.fillRect and ctx.arc. This version replaces the shapes with external image assets (plus-all-open.png and plus-eyes-open.png).

Images are loaded using the Image() constructor.

ctx.drawImage() is used to render these images on the canvas.

Event-Driven Shape (Image) Switching

OLD APPROACH: Shape switching was managed using a boolean variable (showSquare) and keyboard events to determine whether to draw a square or circle.

NEW APPROACH: This behavior remains largely the same, but the rendered shapes are now replaced by corresponding images (img1 for square, img2 for circle).

KEY CONCEPTS: keydown and keyup events still toggle showSquare based on whether the space bar is pressed or released.

The state of showSquare determines which image is drawn using ctx.drawImage.

Image Position and Boundary Control

SHARED POSITION OBJECT: Similar to the previous implementation, both images share the same position (position.x and position.y).

BOUNDRY LOGIC: The update function ensures that the images cannot move outside the canvas.

Boundary logic has been adjusted to account for the image dimensions (e.g., img1Width and img2Height), which replace the square size or circle radius used in the previous version.

Explaining my code

ADVANCED CANVAS ANIMATION

DYNAMIC SPRITE SYSTEM: A spritesheet-like system is implemented for character animation. sprites is an object containing preloaded image assets for different character states (open/closed mouth, blinking, and character toggle).

getSprite() determines the appropriate sprite to render based on the states open, blink and charToggle.

BLINKING MECHANISM: Uses setInterval and setTimeout to toggle blinking every 2.6 seconds for a short duration (100ms). Adds realism by automating character behavior.

Updates are handled through an update() unction, ensuring smooth transitions.

MOVEMENT AND SIZING: Characters can move with WASD keys, resize with the arrow keys, and toggle characters with P and M.

CANVAS RENDERING: The draw() function ensures the correct sprite is drawn based on the character's position, size, and current state. Offset calculations (position.x - offset,etc.) ensure the sprite is centered on the canvas.

Real-Time Updates

The game loop is driven by the render() method, which combines update() and draw() into a continuous rendering cycle using requestAnimationFrame.

This ensures smooth, efficient rendering and animation.

The loop dynamically adjusts the canvas content as the user interacts with the character (movement, size, toggles).
