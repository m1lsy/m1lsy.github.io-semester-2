Layout

Designing my layout using a grid-based approach was a game-changer for positioning elements perfectly on the page. I utilized grid containers for each section of my website, and this method not only streamlined the development process but also ensured a clean, organized structure. As this was my first experience building a website with grids, I found it to be an incredibly efficient way of coding.

One of the standout advantages of this approach was the reusability of the sections I built. Since the layout remained consistent, I could easily repurpose these sections across different projects. This significantly reduced the time and effort required later in the development process. Overall, adopting grids improved both my workflow and the scalability of my designs, and it’s a technique I’ll undoubtedly continue to use in future projects.

Side Bar Navigation

Initially, my portfolio design focused heavily on maintaining a clean and minimal aesthetic, and I hadn’t given much thought to the navigation aspect. I was determined to avoid cluttering the screen with a navigation bar. However, as I started incorporating all of my projects, I realized that navigating the portfolio became difficult for users by scrolling through the portfolio myself.

To address this, I decided to implement a minimal sidebar that seamlessly matched the overall design. This sidebar not only preserved the simplicity of the layout but also provided a convenient way for users to navigate between projects. Additionally, I included a smooth transition effect that hides the sidebar on the hero page, ensuring that the main introduction remains distraction-free.

User Test on navigation

After implementing the sidebar, I conducted a user testing session to evaluate its functionality and ensure smooth navigation throughout the portfolio. The test was designed to simulate real-world interactions and identify any areas for improvement.

Testing Setup and Tasks

I invited a user to complete the following tasks:

1. Navigate from the Hero Page to Project 2
2. Open the Design Process Section in Project 2
3. Navigate from the Design Process Section in Project 2 to the Portfolio Creation Section

Results and Feedback

The overall feedback was positive. The user reported that the navigation felt intuitive and seamless, and the sidebar's minimal design complemented the portfolio's clean aesthetic. However, one area of improvement was identified: when viewing the design process or research sections of a project which open onto a new page, it wasn’t immediately clear which project the user was exploring.

suggestions

The user suggested highlighting the active project number in the sidebar while on seperate pages such as the design or research sections. This small but impactful adjustment would improve clarity and provide users with a stronger sense of orientation.

Improvements Implemented

Based on the feedback, I updated the sidebar to highlight the current project number whenever a user navigates to a subsection like the design or research pages. This enhancement ensures users always know where they are within the portfolio, enhancing both usability and the overall user experience.

By conducting this test, I was able to refine the navigation system and ensure it met the needs of potential users, aligning with my goal of creating a functional and user-friendly portfolio.
